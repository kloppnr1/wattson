# BRS-034 — Request for Prices (Prisanmodning)

> **Status: planned** — BRS-034 support is implemented but not yet activated in production.

BRS-034 allows a market participant to request historical or current price/charge data from DataHub. This is used when WattsOn needs to retrieve prices for a metering point that has not yet received a BRS-031 push (e.g. after onboarding a new customer).

## Message Flow

1. WattsOn sends a price request to DataHub specifying the metering point (GSRN) and date range.
2. DataHub responds with either:
   - **Price data** (D18/D08 format, same as BRS-031) — WattsOn delegates processing to the BRS-031 handler and marks the `BrsProcess` as `Completed`.
   - **Rejection** (RSM-035 with error) — WattsOn marks the `BrsProcess` as `Rejected` and logs the reason.

## DataHub Endpoints

| Environment | Base URL |
|-------------|----------|
| Preprod     | `https://preprod.b2b.datahub3.dk` |
| Production  | `https://b2b.datahub3.dk` |

## Domain Entities

- `BrsProcess` (type: `PrisAnmodning`)
- Downstream: same as [BRS-031](BRS-031.md) (`Price`, `PricePoint`, `PriceLink`)

## State Machine

`Draft → Submitted → Completed` (on data received)
`Draft → Submitted → Rejected` (on DataHub rejection)
