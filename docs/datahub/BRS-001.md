# BRS-001 — Supplier Change (Leverandørskift)

BRS-001 covers the process of transferring a metering point from one electricity supplier to another. WattsOn supports both the **gaining supplier** (initiator) and **losing supplier** (recipient) roles.

## Message Flow

### Initiator (gaining supplier)

1. WattsOn sends a supplier change request to DataHub (CIM JSON, RSM-001 format) with the customer's CPR or CVR number and the desired effective date.
2. DataHub responds with an **RSM-001** confirmation once the change is approved.
3. On confirmation, WattsOn creates a new `Supply` record for the metering point effective on the agreed date.

### Recipient (losing supplier)

1. DataHub sends an **RSM-004** stop-of-supply notification when a customer is moving away.
2. WattsOn records a `BrsProcess` of type `Leverandørskift` (recipient role) and closes the outgoing `Supply` at the effective date.

## Requirements

- Either CPR (personal ID) or CVR (business ID) must be provided for the change request.
- The effective date must be in the future in production environments (past dates are allowed in simulation/preprod).

## DataHub Endpoints

| Environment | Base URL |
|-------------|----------|
| Preprod     | `https://preprod.b2b.datahub3.dk` |
| Production  | `https://b2b.datahub3.dk` |

## Domain Entities

- `BrsProcess` (type: `Leverandørskift`)
- `Supply` — created/closed as a result of the process
- `MeteringPoint` — the metering point being transferred

## State Machine

The BRS-001 state machine transitions through: `Draft → Submitted → Confirmed → Completed` (initiator) or is created directly in `Completed` state (recipient, after the stop notification is processed).

Rejection results in status `Rejected`.
