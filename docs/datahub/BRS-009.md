# BRS-009 — Move-In / Move-Out (Tilflytning / Fraflytning)

BRS-009 covers metering point occupancy changes: a new customer moving in (`Tilflytning`) or a customer moving out (`Fraflytning`). These events are initiated by or notified to WattsOn via DataHub 3.0.

## Message Flow

### Move-In (Tilflytning)

1. WattsOn submits a move-in request to DataHub for a given GSRN and effective date.
2. DataHub responds with an **RSM-001** confirmation.
3. On confirmation, the `BrsProcess` is marked as `Completed` and the new `Supply` record becomes active.

### Move-Out (Fraflytning)

1. DataHub sends an **RSM-004** notification indicating that a customer is moving out of a metering point.
2. WattsOn closes the current `Supply` at the given effective date and records a `BrsProcess` of type `Tilflytning` (move-out variant).

## DataHub Endpoints

| Environment | Base URL |
|-------------|----------|
| Preprod     | `https://preprod.b2b.datahub3.dk` |
| Production  | `https://b2b.datahub3.dk` |

## Domain Entities

- `BrsProcess` (type: `Tilflytning`)
- `Supply` — created (move-in) or closed (move-out)
- `MeteringPoint` — identified by GSRN

## State Machine

Move-in: `Draft → Submitted → Confirmed → Completed`

Move-out: Process is created directly in `Completed` state when the DataHub notification is processed.
